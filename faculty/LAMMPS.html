LAMMPS Tools

This page includes several tools that I have written for use with LAMMPS that I want to contribute back to the general community. These tools come with no warranty or promises of any kind—my only hope is that people will find them useful.
LAMMPS.F90
This file (and others that accompany it) is now included in the LAMMPS distribution under examples/COUPLE/fortran2
The goal is an improved Fortran interface to LAMMPS. Specifically, it allows you to couple Fortran programs to LAMMPS using an interface very similar to the C library interface to LAMMPS. An example file (simple.f90) is included, but the intent is that the Fortran interface will be essentially the same as the C interface, using Fortran’s ISO_C_binding intrinsic module.
Instructions are given in the README file that accompanies the interface. The biggest difference from the C interface is that any pointers that are returned by the C interface are instead accepted as the first argument of a subroutine by the Fortran interface. This is necessary because the type of the return value cannot be determined at compile time.
plot-thermo-data
This shell script processes LAMMPS output (or log files) and plots it in Grace (xmgrace)
Usage:
  • To get help:
    plot-thermo-data -h
  • Plot ALL columns throughout the entire run:
    plot-thermo-data [outputfile]
  • Plot only the temperature against the time step:
    plot-thermo-data -f Temp [outputfile]
  • Plot only the pressure from the 3rd block (corresponding to run or minimize commands) against the time step:
    plot-thermo-data -f Press -s 3 [outputfile]
lammps.vim
This is an enhanced syntax highlighting file that attempts to give you some warning when you make a mistake in your LAMMPS input files. It is very difficult to keep this up to date, as it requires me to edit it every time anyone adds an option to any of the commands.
The ComputeFrenkel and DumpFrenkel classes.
These have been published in Computer Physics Communications; the source code is available as Supporting Information.
These classes allow you to identify defects (Frenkel pairs) in cascade damage simulations. The classes allow you both to compute the number, size, and location of these defects; and to view the damage created over time without having to store the position of every atom in the system and post-process the results.
Region style ellipsoid
A region style that creates ellipsoidal regions. This was originally written for a paper on oblate spheroidal bubbles—which, oddly enough, I was not a co-author of—but is not part of the LAMMPS distribution at present.
Syntax:
region [ID] ellipsoid x y z a b c
where x, y, and z are the coordinates of the center of the ellipse and a, b, and c are the axes of the ellipsoid.
To use this, place the following files in the src directory before building LAMMPS:
  • region_ellipsoid.h
  • region_ellipsoid.cpp
